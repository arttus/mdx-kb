# Code Blocks

MDX Knowledge Base provides powerful code block features with syntax highlighting powered by Shiki.

## Basic Code Blocks

Create a code block by wrapping your code in triple backticks:

```javascript
function hello() {
  console.log("Hello, World!");
}
```

## Language Support

Specify the language for proper syntax highlighting:

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

const user: User = {
  id: 1,
  name: "John Doe",
  email: "john@example.com"
};
```

```python
def greet(name):
    return f"Hello, {name}!"

print(greet("World"))
```

```rust
fn main() {
    let message = "Hello, World!";
    println!("{}", message);
}
```

## File Names

Add a title to show the filename:

```typescript title="user.ts"
export interface User {
  id: number;
  name: string;
  email: string;
}

export function createUser(name: string, email: string): User {
  return {
    id: Math.random(),
    name,
    email
  };
}
```

## Line Highlighting

Highlight specific lines using `{lineNumbers}`:

```javascript title="app.js" {2,4-6}
function processData(data) {
  // This line is highlighted
  const result = data.map(item => {
    // These lines are highlighted
    return item * 2;
  });
  
  return result;
}
```

## Inline Code

Use single backticks for `inline code` within text.

## Best Practices

1. **Always specify the language** for proper highlighting
2. **Add file names** when showing file contents
3. **Highlight important lines** to draw attention
4. **Keep examples concise** and focused
5. **Test your code** before documenting it
