# Writing MDX

MDX is a powerful format that combines Markdown with JSX, allowing you to use React components in your documentation.

## What is MDX?

MDX stands for **Markdown + JSX**. It lets you write JSX directly in your Markdown files, making your documentation interactive and dynamic.

## Basic Syntax

### Headings

```markdown
# Heading 1
## Heading 2
### Heading 3
#### Heading 4
```

### Text Formatting

You can use **bold**, *italic*, ~~strikethrough~~, and `inline code`.

### Lists

Unordered lists:

- Item 1
- Item 2
  - Nested item
  - Another nested item
- Item 3

Ordered lists:

1. First item
2. Second item
3. Third item

### Links and Images

Create links: [MDX Knowledge Base](/)

### Blockquotes

> This is a blockquote. Use it to highlight important information or quotes.

## Frontmatter

Every MDX file should start with frontmatter:

```yaml
---
title: "Page Title"
description: "Page description"
date: "2024-01-15"
category: "guides"
tags: ["mdx", "documentation"]
---
```

<Callout type="info">
  Frontmatter is used for metadata and helps with SEO and organization.
</Callout>

## Best Practices

1. **Use descriptive headings** - Make your content scannable
2. **Add code examples** - Show, don't just tell
3. **Include visuals** - Diagrams and images help understanding
4. **Keep it concise** - Break long content into smaller sections
5. **Use callouts** - Highlight important information
