# Theming

Customize colors, fonts, and styles to match your brand.

## Color System

MDX Knowledge Base uses CSS variables for theming. All colors are defined in `src/app/globals.css`:

```css title="src/app/globals.css"
@theme {
  --color-background: oklch(100% 0 0);
  --color-foreground: oklch(9% 0.005 285.824);
  --color-primary: oklch(47.6% 0.113 264.4);
  --color-secondary: oklch(96.1% 0.004 285.824);
  --color-muted: oklch(96.1% 0.004 285.824);
  --color-accent: oklch(96.1% 0.004 285.824);
  --color-destructive: oklch(57.6% 0.242 27.325);
  --color-border: oklch(89.8% 0.004 285.824);
}
```

## Dark Mode

Dark mode colors are defined separately:

```css
@media (prefers-color-scheme: dark) {
  @theme {
    --color-background: oklch(9% 0.005 285.824);
    --color-foreground: oklch(98% 0.005 285.824);
    --color-primary: oklch(70.6% 0.113 264.4);
    /* ... more colors */
  }
}
```

## Custom Colors

Add new colors to your theme:

```css
@theme {
  --color-brand: oklch(60% 0.15 250);
  --color-success: oklch(65% 0.15 145);
  --color-warning: oklch(75% 0.15 85);
  --color-error: oklch(60% 0.20 25);
}
```

Then use them in your components:

```tsx
<div className="bg-(--color-brand) text-white">
  Custom branded section
</div>
```

## Typography

Customize fonts in the theme:

```css
@theme {
  --font-sans: ui-sans-serif, system-ui, sans-serif;
  --font-mono: ui-monospace, 'Cascadia Code', monospace;
}
```

Import custom fonts in `src/app/layout.tsx`:

```typescript
import { Inter, JetBrains_Mono } from 'next/font/google';

const inter = Inter({ subsets: ['latin'] });
const jetbrains = JetBrains_Mono({ subsets: ['latin'] });
```

## Border Radius

Adjust border radius values:

```css
@theme {
  --radius-sm: 0.25rem;
  --radius-md: 0.375rem;
  --radius-lg: 0.5rem;
  --radius-xl: 0.75rem;
}
```

## Component Styling

Override component styles using Tailwind classes:

```tsx
<Button className="bg-brand hover:bg-brand/90">
  Custom Button
</Button>
```

## Code Block Themes

Customize syntax highlighting themes in `next.config.ts`:

```typescript
rehypePrettyCode, {
  theme: {
    dark: 'github-dark',
    light: 'github-light',
  },
}
```

Available themes:
- `github-dark` / `github-light`
- `dracula`
- `nord`
- `monokai`
- `one-dark-pro`
- And many more!

## Prose Styling

Customize MDX content styling in `src/app/globals.css`:

```css
.prose h1 {
  font-size: 2.25rem;
  font-weight: 800;
  margin-bottom: 1rem;
}

.prose p {
  margin-bottom: 1rem;
  line-height: 1.75;
}
```

<Callout type="info">
  All theme changes are automatically applied to both light and dark modes.
</Callout>

## Best Practices

1. **Use CSS variables** - Makes theme switching seamless
2. **Test both modes** - Ensure good contrast in light and dark
3. **Keep it accessible** - Maintain WCAG AA contrast ratios
4. **Be consistent** - Use the same color palette throughout
5. **Document changes** - Keep track of custom theme modifications

## Example: Custom Brand Theme

```css title="src/app/globals.css"
@theme {
  /* Brand Colors */
  --color-brand-primary: oklch(55% 0.20 260);
  --color-brand-secondary: oklch(70% 0.15 200);
  
  /* Override defaults */
  --color-primary: var(--color-brand-primary);
  --color-accent: var(--color-brand-secondary);
  
  /* Custom radius */
  --radius-lg: 1rem;
}
```

<Callout type="success">
  Your MDX Knowledge Base now has a custom theme!
</Callout>
